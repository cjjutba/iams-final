# Output Contract

Every AI implementation task for MOD-10 must return:

1. Scope
- Implemented `FUN-10-*` IDs
- Explicitly excluded items

2. Changes
- Files created/modified with short rationale

3. Validation
- Tests executed (`T10-*`) and summary
- Manual verification steps completed

4. Contract Compliance
- Confirmation consumed API contracts were followed
- Confirmation faculty screen IDs and UI-state requirements were implemented

5. Risks and Follow-Ups
- Known risks
- Docs or design decisions requiring follow-up
