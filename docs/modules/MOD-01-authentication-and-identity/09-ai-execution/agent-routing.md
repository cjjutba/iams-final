# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| Auth security/token logic | `auth-security-specialist.md` | `backend-core-specialist.md` |
| API endpoint scaffolding | `backend-core-specialist.md` | `business-logic-specialist.md` |
| Mobile auth integration | `mobile-api-integration.md` | `mobile-state-manager.md` |
| Form validation | `mobile-forms-validator.md` | `mobile-frontend-specialist.md` |
| Testing automation | `test-automation-specialist.md` | `backend-core-specialist.md` |
| Documentation sync | `docs-writer.md` | `backend-core-specialist.md` |

## Routing Rules
- Security-sensitive changes must include auth-security review.
- API contract changes require docs update before implementation.
- UI flow changes require screen docs and state matrix update.
