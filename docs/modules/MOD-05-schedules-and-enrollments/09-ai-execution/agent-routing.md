# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| Schedule endpoint scaffolding | `backend-core-specialist.md` | `business-logic-specialist.md` |
| Validation/business rules | `business-logic-specialist.md` | `database-specialist.md` |
| Enrollment joins and constraints | `database-specialist.md` | `backend-core-specialist.md` |
| Mobile schedule integration | `mobile-api-integration.md` | `mobile-state-manager.md` |
| Testing automation | `test-automation-specialist.md` | `backend-core-specialist.md` |
| Documentation sync | `docs-writer.md` | `backend-core-specialist.md` |

## Routing Rules
- API contract changes require docs update before implementation.
- Time semantics changes require dependency module review.
- Roster query changes require enrollment constraint tests.
