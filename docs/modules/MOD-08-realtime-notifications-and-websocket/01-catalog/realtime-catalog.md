# Realtime Catalog

## Module Summary
- Module ID: `MOD-08`
- Module Name: Realtime Notifications and WebSocket
- Primary Domain: Backend + Mobile

## Auth Context
- **User-Facing:** FUN-08-01 (WebSocket connect) and FUN-08-05 (reconnect/cleanup) — require Supabase JWT.
- **System-Internal:** FUN-08-02, FUN-08-03, FUN-08-04 — backend service functions, no JWT required.

## Function Catalog
| Function ID | Function Name | Type | Auth | Brief Description |
|---|---|---|---|---|
| FUN-08-01 | Open authenticated WebSocket connections | User-facing | Supabase JWT (query param `token`) | Accept socket only for authenticated user context. |
| FUN-08-02 | Publish attendance update events | System-internal | None (service call) | Push attendance status changes to relevant clients. |
| FUN-08-03 | Publish early-leave events | System-internal | None (service call) | Push early-leave alerts generated by presence logic. |
| FUN-08-04 | Publish session-end summary events | System-internal | None (service call) | Push class-end summary payload to clients. |
| FUN-08-05 | Handle reconnect and stale cleanup | User-facing | Supabase JWT (reconnect) | Recover from disconnect and clean stale connection state. |

## Event Catalog
| Event Type | Producer | Consumer Screens |
|---|---|---|
| `attendance_update` | Attendance service (`MOD-06`) via notification service | SCR-021, SCR-029 |
| `early_leave` | Presence service (`MOD-07`) via notification service | SCR-021, SCR-025, SCR-029 |
| `session_end` | Session finalization flow via notification service | SCR-018, SCR-021, SCR-029 |

## Actors
- **Student (authenticated):** Connects to own WebSocket, receives session-end summaries and routed notifications.
- **Faculty (authenticated):** Connects to own WebSocket, receives attendance updates, early-leave alerts, session-end summaries.
- **Admin (authenticated):** Same as faculty for WebSocket access.
- **Presence service (MOD-07):** Invokes FUN-08-03 to publish early-leave events.
- **Attendance service (MOD-06):** Invokes FUN-08-02 to publish attendance updates.

## API Surface
- `WS /ws/{user_id}` — Supabase JWT required (query param `token`)
- Base URL: `ws://localhost:8000/api/v1` (development), `wss://` in production
