# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| Edge runtime pipeline | `edge-device-specialist.md` | `edge-api-specialist.md` |
| Payload schema and sender | `edge-api-specialist.md` | `backend-core-specialist.md` |
| Queue/retry reliability | `edge-device-specialist.md` | `test-automation-specialist.md` |
| Deployment/runtime service setup | `devops-deployment.md` | `edge-device-specialist.md` |
| Documentation sync | `docs-writer.md` | `edge-device-specialist.md` |

## Routing Rules
- API contract changes require docs update before implementation.
- Queue policy changes require test plan updates.
- Recovery behavior changes require deployment/runtime docs update.
