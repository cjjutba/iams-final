# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| presence state engine | `tracking-presence-specialist.md` | `business-logic-specialist.md` |
| endpoint scaffolding | `backend-core-specialist.md` | `tracking-presence-specialist.md` |
| threshold/score rules | `business-logic-specialist.md` | `tracking-presence-specialist.md` |
| faculty UI integration | `mobile-api-integration.md` | `mobile-state-manager.md` |
| testing automation | `test-automation-specialist.md` | `tracking-presence-specialist.md` |
| documentation sync | `docs-writer.md` | `backend-core-specialist.md` |

## Routing Rules
- Threshold/interval changes require test and config updates.
- API contract changes require docs update before implementation.
- Event behavior changes require alignment with websocket module.
