# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| User endpoint scaffolding | `backend-core-specialist.md` | `business-logic-specialist.md` |
| Validation/business rules | `business-logic-specialist.md` | `database-specialist.md` |
| Lifecycle/delete safeguards | `database-specialist.md` | `backend-core-specialist.md` |
| Mobile profile integration | `mobile-api-integration.md` | `mobile-state-manager.md` |
| UI profile screens | `mobile-frontend-specialist.md` | `mobile-forms-validator.md` |
| Testing automation | `test-automation-specialist.md` | `backend-core-specialist.md` |
| Documentation sync | `docs-writer.md` | `backend-core-specialist.md` |

## Routing Rules
- API contract changes require docs update before implementation.
- Lifecycle strategy changes must include database review.
- UI flow changes require screen docs and state matrix update.
