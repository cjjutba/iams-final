# Agent Routing

## Recommended Agent Assignment
| Work Type | Primary Agent | Secondary Agent |
|---|---|---|
| ML/embedding and recognition logic | `ml-face-recognition.md` | `backend-core-specialist.md` |
| API endpoint scaffolding | `backend-core-specialist.md` | `business-logic-specialist.md` |
| Edge recognition integration checks | `edge-api-specialist.md` | `edge-device-specialist.md` |
| Mobile face capture integration | `mobile-camera-face-capture.md` | `mobile-api-integration.md` |
| Testing automation | `test-automation-specialist.md` | `ml-face-recognition.md` |
| Documentation sync | `docs-writer.md` | `backend-core-specialist.md` |

## Routing Rules
- Any threshold/model change requires ML and backend review.
- API contract changes require docs update before implementation.
- Capture flow changes require screen docs and state matrix update.
