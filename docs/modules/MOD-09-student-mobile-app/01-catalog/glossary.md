# Glossary

- Auth stack: unauthenticated navigation path before student session is established.
- Student stack: authenticated student navigation path.
- Registration draft: temporary local data captured across registration steps.
- Token persistence: secure local storage and restore of auth tokens.
- Session restore: loading persisted credentials to re-authenticate app state.
- Re-registration: replacing existing face registration with fresh captures.
- State hydration: loading cached state into runtime store on app start.
- UI state triad: loading, empty, and error behavior for data-driven screens.
