# Demo Checklist

## Pre-Demo Setup
- [ ] Mobile app has correct API/WS environment values.
- [ ] Backend services for auth, face, schedule, attendance, websocket are running.
- [ ] Student test account and/or registration data source available.

## Demo Steps
- [ ] Show onboarding and welcome routing.
- [ ] Demonstrate student login/session restore.
- [ ] Demonstrate 4-step registration flow with validations.
- [ ] Open student home/history/detail and verify data states.
- [ ] Demonstrate profile edit and face re-registration path.
- [ ] Show student notifications and reconnect behavior.

## Pass Criteria
- [ ] Student flow is usable end-to-end for MVP.
- [ ] Validation gates block invalid registration progression.
- [ ] Notifications update without app restart after reconnect.
